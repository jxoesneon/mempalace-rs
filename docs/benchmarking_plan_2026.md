# Implementation Plan: 2026 Gold Standard Benchmarking for MemPalace-RS

## 1. Objective

To replace the flawed legacy benchmarking suite (LoCoMo, LongMemEval) with a rigorous, high-integrity implementation of the **2026 Gold Standards**: **RULER**, **BABILong**, **BEAM**, and **StructMemEval**. This ensures `mempalace-rs` is validated against multi-hop reasoning, 10M+ token persistence, and complex structural organization without the "benchmaxx" shortcuts of the original project.

## 2. Core Principles (Anti-Fraud Methodology)

- **No top_k Bypass:** All retrieval benchmarks MUST use `top_k <= 10` for standard evaluation. `top_k=50` is strictly for diagnostic/ablation use and must be labeled as such.
- **End-to-End QA:** Accuracy is measured by the final answer generated by a "Judge LLM" (GPT-4o/Claude-3.5-Sonnet) comparing retrieved context to ground truth, NOT just Recall@K.
- **Corpus Scaling:** Benchmarks MUST exceed the native context window (test at 512k, 1M, and 10M tokens).
- **No Hardcoded Patches:** Remove all "person-name boosts" or dataset-specific regex filters. Use semantic entity resolution (canonical IDs) instead.
- **Transparent Ingestion:** Document exactly how memories enter the system (batch vs. streaming) and any preprocessing (AAAK compression).

## 3. Benchmark Integration Strategy

### Phase A: RULER (Multi-Needle & Aggregation)

- **Goal:** Validate multi-hop tracing and semantic discrimination.
- **Implementation:**
  - Create `src/benchmarks/ruler.rs`.
  - Implement "Variable Tracking" tasks: Trace values of variables across 1M tokens.
  - Implement "Aggregation" tasks: Count specific entity occurrences.
  - Metric: nDCG and multi-needle recall.

### Phase B: BABILong (Reasoning-in-a-Haystack)

- **Goal:** Test 1M to 10M token reasoning limits.
- **Implementation:**
  - Create `src/benchmarks/babilong.rs`.
  - Port all 20 bAbI reasoning tasks.
  - Use PG-19 (Project Gutenberg) as the background "haystack" corpus.
  - Test at 1M and 10M token splits.

### Phase C: BEAM (Agentic Memory & Coherence)

- **Goal:** Evaluate narrative logic and persona consistency.
- **Implementation:**
  - Create `src/benchmarks/beam.rs`.
  - Implement "Nugget-based" evaluation (atomic semantic units).
  - Integrate bidirectional interaction modules (simulated User/Assistant).
  - Metric: Mean Nugget Score (0.0 to 1.0).

### Phase D: StructMemEval (Organizational Prowess)

- **Goal:** Test Trees, States, and Ledgers.
- **Implementation:**
  - Create `src/benchmarks/struct_mem.rs`.
  - Implement "Debt Settlement" (Ledger) and "Hierarchy Traversal" (Tree) tasks.
  - Include "Memory Organization Hints" toggle to isolate strategic vs. execution failures.
  - Metric: Structural Accuracy and State Consistency.

## 4. Technical Infrastructure Changes

- **Harness Refactor:** Move from `src/benchmark.rs` to a modular `src/benchmarks/mod.rs` with trait-based benchmark definitions.
- **Judge LLM Integration:** Add a `Judge` trait and an OpenAI/Anthropic client for automated E2E evaluation.
- **Metrics Collector:** A unified `MetricsStore` to record time-to-retrieve, token usage, and accuracy across all tests.
- **Artifacts:** Automated generation of `bench_report.json` and `bench_report.md`.
